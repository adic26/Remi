Instruments at Design-Time (or if the -localDomain switch is used to launch the application):

The instrument xml definitions must be converted into classes using the InstrumentClassGenerator custom tool (right-click the xml file and select Properties, then enter InstrumentClassGenerator into the Custom Tool property). Generated classes must have the Build Action property set to Compile (this will be set by default when they are converted from xml). The instrument classes will be compiled into the main application assembly. Using the -localDomain switch prevents run-time modification of instrument definitions - definitions are baked into the application at design-time.

 

Instrument at Run-Time:

The instrument xml definitions must have the Copy To Output Directory property set to either Copy If Newer or Copy Always. They will then be copied to the Instruments folder in the output directory, where they will be picked up by the run-time compiler and built into the dynamic sequence assembly when the test sequence is executed.

 

Test Sequence Design-Time/Run-Time Synchronization

The test sequence classes located in the Sequences folder are synchronized with the Sequence configuration files in a post-build event. This allows any design-time changes to the sequence classes to be automatically pushed to the run-time configuration objects. Also, if any new sequences are added at run-time, they will be parsed into C# code files and added to the design-time Sequences folder. Modifying a sequence at run-time DOES NOT overwrite or update existing design-time classes in the Sequences folder.

 

Sequences normal operation (using separate AppDomain):

Sequence objects are dynamically compiled using the source code contained in the Sequence configuration objects.

 

Sequences when the -localDomain switch is used to launch the application:

Sequences are compiled into the main application assembly at design-time. Using the -localDomain switch prevents run-time modification of test sequences - sequences are baked into the application at design-time.
