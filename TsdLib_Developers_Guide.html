Instrument  files are not compiled as part of the build process - the xml source files are copied to the output directory where they can be picked up by the code generator and dynamically compiled into a separate Application Domain where the test sequence will execute.

There is a pre-build event to transform the instrument xml files into the .NET class file, but this is just to show the developer what the generated instrument classes look like.

 

 

Sequence files are not compiled as part of the build process. There is a post-build event to upload the contents of the sequence files to the database as Sequence config items. This makes the design-time sequences available in the run-time sequence selector.
